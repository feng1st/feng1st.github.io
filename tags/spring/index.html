Spring DispatcherServlet相关知识点

Posted on June 25, 2017

DispatcherServlet作用 作为Spring MVC的集中访问点 通过HandlerMapping，将请求映射到Handler 返回一个HandlerExecutionChain，包含一个HandlerAdaptor和多个HandlerInterceptor 通过不同的HandlerAdaptor，支持不同的处理器 通过ViewResolver，支持到具体视图的解析 通过HandlerExceptionResolver，实现对异常的处理 每一部分都可以很方便的扩展 DispatcherServlet流程 请求被DispatcherServlet截获 DispatcherServlet根据handlerMapping，获得一个HandlerExecutionChain，然后获得HandlerAdaptor HandlerAdaptor实例化一个ServletInvocableHandlerMethod，对应具体Contrller的一个方法 ServletInvocableHandlerMethod.invokeAndHandle()分两步进行处理 Object returnValue = invokeForRequest(...); this.returnValueHandlers.handleReturnValue(returnValue, ...); 处理请求主要是支持对不同类型的参数的解析 通过argumentResolvers实现 在父类InvocableHandlerMethod定义 是HandlerMethodArgumentResolverComposite类 是HandlerMethodArgumentResolver的组合 处理响应主要是对不同返回值做不同的转换 通过returnValueHandlers实现 是HandlerMethodReturnValueHandlerComposite类 是HandlerMethodReturnValueHandler的组合 argumentResolvers和returnValueHandlers是在实例化ServletInvocableHandlerMethod时，从RequestMappingHandlerAdaptor传入的 RequestMappingHandlerAdaptor的这两个属性，是在实例化时，从Spring容器依赖注入的 这两个属性默认值就包含了各种常见的HandlerMethodArgumentResolver和HandlerMethodReturnValueHandler 这就是为什么Controller的方法自动支持包括@RequestBody, @RequestParam，@PathValue, 对象, WebRequest, ServletRequst, HttpSession, InputStream, Reader, ServletResponse, OutputStream, Writer等类型的参数
