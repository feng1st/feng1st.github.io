Spring DispatcherServlet相关知识点

Posted on June 25, 2017
Service
DispatcherServlet, Spring

DispatcherServlet作用

  1. 作为Spring MVC的集中访问点
  2. 通过HandlerMapping，将请求映射到Handler
    • 返回一个HandlerExecutionChain，包含一个HandlerAdaptor和多个HandlerInterceptor
  3. 通过不同的HandlerAdaptor，支持不同的处理器
  4. 通过ViewResolver，支持到具体视图的解析
  5. 通过HandlerExceptionResolver，实现对异常的处理

每一部分都可以很方便的扩展

DispatcherServlet流程

  1. 请求被DispatcherServlet截获
  2. DispatcherServlet根据handlerMapping，获得一个HandlerExecutionChain，然后获得HandlerAdaptor
  3. HandlerAdaptor实例化一个ServletInvocableHandlerMethod，对应具体Contrller的一个方法
  4. ServletInvocableHandlerMethod.invokeAndHandle()分两步进行处理
    1. Object returnValue = invokeForRequest(...);
    2. this.returnValueHandlers.handleReturnValue(returnValue, ...);
  5. 处理请求主要是支持对不同类型的参数的解析
    • 通过argumentResolvers实现
    • 在父类InvocableHandlerMethod定义
    • 是HandlerMethodArgumentResolverComposite类
    • 是HandlerMethodArgumentResolver的组合
  6. 处理响应主要是对不同返回值做不同的转换
    • 通过returnValueHandlers实现
    • 是HandlerMethodReturnValueHandlerComposite类
    • 是HandlerMethodReturnValueHandler的组合
  7. argumentResolvers和returnValueHandlers是在实例化ServletInvocableHandlerMethod时，从RequestMappingHandlerAdaptor传入的
  8. RequestMappingHandlerAdaptor的这两个属性，是在实例化时，从Spring容器依赖注入的
  9. 这两个属性默认值就包含了各种常见的HandlerMethodArgumentResolver和HandlerMethodReturnValueHandler
    • 这就是为什么Controller的方法自动支持包括@RequestBody, @RequestParam，@PathValue, 对象, WebRequest, ServletRequst, HttpSession, InputStream, Reader, ServletResponse, OutputStream, Writer等类型的参数
  10. 例子：RequestResponseBodyMethodProcessor
    • 用来处理@RequestBody和@ResponseBody
    • 同时实现了HandlerMethodArgumentResolver和HandlerMethodReturnValueHandler接口
    • 内部实现主要是读写Request和Response的Body，和使用HttpMessageConverter转换数据

Filter和HandlerInterceptor

总体来说，两者是类似的。不同点：

  1. Filter在web.xml定义，HandlerInterceptor在application context里定义
  2. HandlerInterceptor接口分得更细
  3. Filter一般用来处理请求内容和视图内容，比如压缩和分块
  4. HandlerInterceptor用来处理一般性面向切面逻辑，比如授权

public interface HandlerInterceptor {
    boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception;
    void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception;
    void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception;
}
